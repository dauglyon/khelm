# Integration: app-shell -> workspace

## Imports Required

### Layout Slot
- The workspace masonry grid renders inside the **Main Workspace** region (fills remaining height below header + toolbar, scrollable, 24px horizontal padding)
- `MainWorkspace` component in `src/features/sessions/components/MainWorkspace.tsx` provides the container element
- The workspace's `@tanstack/react-virtual` virtualizer uses the main workspace element as its scroll container (not `window`) -- the workspace domain must receive a ref to or query for this container
- The workspace must respect the main workspace's padding and overflow behavior

### Session Context
- **`sessionStore.activeSessionId`** -- the workspace scopes all card queries and state to the active session
- **`useSession(sessionId)`** hook -- the workspace may need session metadata (e.g., member list for cross-card reference resolution)
- The workspace must not render card content when `activeSessionId` is null

### API Layer (Orval + MSW)
- Card list/fetch endpoint types and hooks generated by Orval from the OpenAPI spec
- TanStack Query hooks for fetching persisted cards (stale-while-revalidate pattern for session cards)
- MSW handlers mock card endpoints during development, providing realistic card data for the masonry grid

### Auth Context
- **`authStore.token`** from `src/common/stores/authStore.ts` -- all card data API requests require authentication
- The workspace renders only within `RequireAuth` + `RequireSession` guarded routes

### Environment Configuration
- **`getApiBaseUrl()`** from `src/common/utils/env.ts` -- base URL for card data fetching
- **`getWsUrl()`** from `src/common/utils/env.ts` -- WebSocket URL for receiving real-time card events (`card:created`, `card:updated`) that trigger workspace store updates (used by the collaboration domain, but the workspace reacts to the resulting state changes)

### Provider Infrastructure
- **TanStack Query** `QueryClientProvider` -- the workspace uses TanStack Query for server-fetched card data (caching, refetch, stale-while-revalidate)
- **LazyMotionProvider** -- card enter animations (`motion.div` with `initial`/`animate`) and detail transitions (`layoutId`) require Motion in the provider tree

### Layout Store
- **`layoutStore.sidebarOpen`** from `src/common/stores/layoutStore.ts` -- when the sidebar opens/closes, the main workspace width changes; the masonry grid must recalculate column count via `ResizeObserver` on its container (this happens naturally if using `ResizeObserver`, but the workspace should not hardcode widths)

## Acceptance Criteria

1. Masonry grid renders inside the Main Workspace layout region at `/session/:id`
2. Virtualizer scroll container is the Main Workspace element, not `window`
3. Workspace card queries are scoped to `activeSessionId` from the session store
4. Workspace does not attempt to render or fetch cards when `activeSessionId` is null
5. Card data fetching uses Orval-generated TanStack Query hooks -- no hand-written fetch calls
6. Card enter animations work correctly (Motion provider is available in the tree)
7. `layoutId` detail transitions work between the masonry grid card and the detail overlay
8. Masonry grid responds to sidebar open/close by recalculating column count (via `ResizeObserver`, not by reading sidebar state directly)
9. MSW mocks provide realistic card data for development and testing
10. Auth token is included in all card data API requests
