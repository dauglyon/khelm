# Integration: app-shell -> data-ingest

## Imports Required

### Layout Slots
- **Toolbar region** -- the data-ingest domain renders a global upload indicator in the toolbar showing the count of active uploads and aggregate progress (see data-ingest.md section 6, "Progress UI", "Global indicator" row)
- The global indicator must fit within the 64px toolbar height and coexist with the input surface
- **Main Workspace region** -- the `WorkspaceDropZone` wraps the main workspace container element; file drops create preview cards in the workspace

### Session Context
- **`sessionStore.activeSessionId`** -- ingested files are associated with the active session; the upload metadata and created cards reference the session ID
- The drop zone and upload flow must not activate when `activeSessionId` is null

### API Layer (Orval + MSW)
- Card creation endpoint types (data-ingest card type) generated by Orval from the OpenAPI spec
- The preview card creation uses the same card creation mutation as other card types, with `type: 'data_ingest'`
- MSW handlers mock the card creation endpoint for data-ingest cards during development
- Future: upload completion callback endpoint types may be added to the OpenAPI spec

### Auth Context
- **`authStore.token`** from `src/common/stores/authStore.ts` -- tus upload requests include the auth token as a header for server-side authorization
- Card creation API calls for preview cards require authentication

### Environment Configuration
- **`getApiBaseUrl()`** from `src/common/utils/env.ts` -- base URL for card creation and upload-related API endpoints
- The tus upload endpoint URL is derived from or configured alongside the API base URL

### Provider Infrastructure
- **TanStack Query** `QueryClientProvider` -- the data-ingest domain uses TanStack Query for:
  - Card creation mutations (creating the preview card on the workspace)
  - Cache invalidation after upload completes (so the workspace reflects the finalized card)
- **LazyMotionProvider** -- drop zone visual feedback animations (border transitions on drag states) may use Motion

### Route Context
- The data-ingest domain operates only within the workspace view at `/session/:id`, inside the `RequireAuth` + `RequireSession` guarded route tree
- The drop zone should not be active on non-workspace pages (home, login, etc.)

## Acceptance Criteria

1. Global upload indicator renders in the Toolbar region without exceeding the 64px height
2. Global upload indicator coexists with the input surface in the toolbar
3. `WorkspaceDropZone` wraps the Main Workspace container element for file drop detection
4. Dropped files create preview cards scoped to `activeSessionId` from the session store
5. Drop zone and upload flow are inactive when `activeSessionId` is null
6. Preview card creation uses Orval-generated types for the data-ingest card type
7. tus upload requests include the auth token from `authStore.token`
8. Card creation after upload completion invalidates TanStack Query card list cache
9. Upload endpoint URL is derived from `getApiBaseUrl()` or a related env configuration
10. Drop zone is only active on the workspace page (`/session/:id`), not on unauthenticated or non-session pages
