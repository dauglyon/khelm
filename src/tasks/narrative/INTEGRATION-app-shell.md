# Integration: app-shell -> narrative

## Imports Required

### Layout Slots
- **Toolbar region** -- the `CompositionToolbar` renders inside the toolbar area, providing the "Compose" button to enter composition mode, "Exit" to leave, "Select All" / "Clear" bulk actions, and a selected card count badge
- The composition toolbar elements must fit within the 64px toolbar height and coexist with the input surface (or replace it contextually when composition mode is active)
- **Sidebar region** -- the composition panel could alternatively render in the sidebar slot (320px collapsible right panel) depending on layout decisions; the architecture doc describes it as a "side panel" at ~50% viewport width, which is larger than the sidebar but uses the same right-side positioning pattern

### Layout Store
- **`layoutStore.sidebarOpen`** and **`layoutStore.setSidebarOpen()`** from `src/common/stores/layoutStore.ts` -- entering composition mode may need to close the sidebar to make room for the composition panel, or the composition panel may replace the sidebar content
- The narrative domain must coordinate with the layout store to avoid conflicts between the sidebar and the composition panel

### Session Context
- **`sessionStore.activeSessionId`** -- narrative export sends `sessionId` in the export payload (`POST /api/narratives/export { sessionId, orderedCardIds, connectiveTexts }`)
- **`useSession(sessionId)`** hook -- provides session metadata for the narrative artifact (e.g., session title as narrative title)
- The narrative domain must not activate when `activeSessionId` is null

### API Layer (Orval + MSW)
- Narrative export endpoint types and hooks generated by Orval from the OpenAPI spec:
  - `POST /api/narratives/export` -- submit composition for export
  - `GET /api/narratives/{id}` -- retrieve exported narrative
  - `GET /api/narratives/{id}/status` -- poll async export status
- MSW handlers mock all narrative endpoints during development, simulating async export with poll-based completion
- Card query hooks (from Orval) for reading card data to display in the composition panel summaries and preview

### Auth Context
- **`authStore.token`** from `src/common/stores/authStore.ts` -- narrative export and retrieval API requests require authentication

### Environment Configuration
- **`getApiBaseUrl()`** from `src/common/utils/env.ts` -- base URL for narrative export and retrieval endpoints

### Provider Infrastructure
- **TanStack Query** `QueryClientProvider` -- the narrative domain uses TanStack Query for:
  - Narrative export mutation
  - Export status polling (`refetchInterval` for async export completion)
  - Card data queries for composition panel card summaries
- **LazyMotionProvider** -- composition panel enter/exit animations, card summary reorder FLIP animations (via Motion `layout` prop on the vertical sortable list), and preview modal transitions

### Route Context
- The narrative domain operates only within the workspace view at `/session/:id`, inside the `RequireAuth` + `RequireSession` guarded route tree
- Composition mode is a UI state change within the workspace, not a separate route

## Acceptance Criteria

1. Composition toolbar elements render in the Toolbar region without exceeding the 64px height
2. Entering composition mode coordinates with the layout store (sidebar closes or composition panel replaces sidebar content)
3. Narrative export payload includes `sessionId` from `activeSessionId` in the session store
4. Narrative API calls (export, status poll, retrieval) use Orval-generated types and hooks
5. MSW handlers mock all three narrative endpoints with realistic responses during development
6. Async export polling uses TanStack Query's `refetchInterval` pattern
7. Composition panel and preview modal animations work correctly (Motion provider is available)
8. Narrative domain is inactive when `activeSessionId` is null
9. All narrative API requests include the auth token from `authStore`
10. Narrative features only activate within `RequireAuth` + `RequireSession` guarded routes at `/session/:id`
