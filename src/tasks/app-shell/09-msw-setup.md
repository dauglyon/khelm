# Task 09: MSW Browser + Node Worker Setup

**ID:** app-shell/09
**Status:** pending
**Deps:** app-shell/08

## Context

This task sets up MSW v2 for both browser (dev server) and Node (Vitest) environments. The browser worker intercepts API requests during development so the frontend can be built without a running backend. The Node server uses the same handlers in Vitest tests. Handlers come from Orval-generated MSW handlers (task 08) with the ability to add custom overrides. The MSW worker is conditionally started in `main.tsx` based on the `VITE_ENABLE_MOCKS` env var.

## Implementation Requirements

### Files to Create/Modify

1. **`src/mocks/browser.ts`** (~10 lines)
   - Import `setupWorker` from `msw/browser`
   - Import aggregated handlers from `./handlers`
   - Export `worker = setupWorker(...handlers)`

2. **`src/mocks/server.ts`** (~10 lines)
   - Import `setupServer` from `msw/node`
   - Import aggregated handlers from `./handlers`
   - Export `server = setupServer(...handlers)`

3. **`src/mocks/handlers.ts`** (~20 lines)
   - Import generated handlers from `src/generated/api/*.msw.ts`
   - Export aggregated handler array: `[...generatedHandlers, ...customHandlers]`
   - Custom handlers array (initially empty) takes precedence (listed last)

4. **`src/main.tsx`** (modify, ~15 lines added)
   - Conditional MSW bootstrap:
     ```
     if VITE_ENABLE_MOCKS is truthy:
       dynamically import ./mocks/browser
       await worker.start({ onUnhandledRequest: 'bypass' })
     then render the app
     ```
   - Use dynamic import so MSW is tree-shaken in production builds

5. **`public/mockServiceWorker.js`** (generated)
   - Run `npx msw init public/` to generate the service worker file

6. **`src/test/setup.ts`** (~20 lines)
   - Import server from `src/mocks/server`
   - `beforeAll(() => server.listen({ onUnhandledRequest: 'error' }))`
   - `afterEach(() => server.resetHandlers())`
   - `afterAll(() => server.close())`

7. **`vitest.config.ts`** or **`vite.config.ts`** test section (~15 lines)
   - Configure Vitest with:
     - `environment: 'happy-dom'`
     - `setupFiles: ['./src/test/setup.ts']`
     - `globals: true`

## Anti-Patterns

- Do NOT import MSW browser worker in test code -- use the Node server
- Do NOT use MSW v1 API (`rest.get`) -- use v2 API (`http.get`)
- Do NOT eagerly import MSW in main.tsx -- use dynamic import for tree-shaking
- Do NOT set `onUnhandledRequest: 'error'` in browser worker -- use `'bypass'` to allow non-API requests through
- Do NOT mock at the fetch/XHR level manually -- MSW handles this at the service worker level
